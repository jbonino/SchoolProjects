Interpreter

Forces


Need a standard way to evaluate language grammar or expression.


Solution


Create an object to interpet the context. Create an abstract class of expressions that defines all methods needed to convert data.
Create concrete class for each expression.

Example (Pseudo Code)

//expecting this syntax "1 dollar to pennies"
// "quantity type to type"
class ConversionContext{
     double quantity;
     string fromType,toType,input; //dollar - pennies
     void ConversionContext(string i){
          input=i;
          quantity = Double.parseDouble(input[0]);
          fromType = string manipulation; //dollar
          toType = string manipulation; //pennies
     }
     getQuantity(){ return quantity; }
     getFromType(){ return fromType; }
     getToType(){ return toType; }
}

abstract class Expression{
     abstact double dollar(double q);
     abstact double quarter(double q);
     abstact double penny(double q);
}

class Dollar extends Expression{
     double dollar(double q){return q;}
     double quarter(double q){return q*4;}
     double penny(double q){return q*100;}
}

class Quarter extends Expression{
     double dollar(double q){return q/4;}
     double quarter(double q){return q;}
     double penny(double q){return q*25;}
}

class Penny extends Expression{
     double dollar(double q){return q/100;}
     double quarter(double q){return q/25;}
     double penny(double q){return q;}
}
