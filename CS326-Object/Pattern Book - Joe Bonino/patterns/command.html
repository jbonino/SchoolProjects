Command

Forces


Issue requests without knowing any logic behind request and who is receiving it.


Solution


Create a object for the request that has everyhing about the command (what command does, an undo). This object is passed to an invoker
which then passes this to a corresponding reciever object that executes the command.


Related Patters


  • Chain of Responsibility

  • Observer


Example (Pseudo Code)

interface Command{
     void execute();
     void undo();
}
public class Television{ //receiver
     void on(){ //turn tv on }
     void off(){ //turn tv off }
}

public turnOn implements Command{ //command
     Television tv;
     public turnOn(Television t){ tv=t; }

     void execute(){ tv.on(); }
     void undo(){ tv.off(); }
}
public turnOff implements Command{ //command
     Television tv;
     public turnOn(Television t){ tv=t; }

     void execute(){ tv.off(); }
     void undo(){ tv.on(); }
}

public class button{ //invoker
     Command command;
     public button(Command c){ command=c; }

     void press(){ command.execute(); }
     void undo(){ command.undo(); }
}
