Observer

Forces


Object has a one-to-many relationship with other objects.


Solution


Object holds a list of dependents and lets them know when ever it changes.


Related Patters


  • Command
  • Chain of Responsibility


Example (Pseudo)


//a realGUI is an existing object that has a 'logic' layer and a 'log' (to save all changes) as dependents.
//realGUI asks user a question and if its right then they win.

class realGUI(){
     List <Observer> dependents;
     //when ever GUI is manipulated by user, run this
     stateChanged(changedstate s){
          for all dependents{
               update(s)
          }
     }
}

class Observer{
     update(changed state s){
          abstract update(s);
     }
}
